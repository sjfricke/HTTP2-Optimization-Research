HTTP2 Optimization

Research conducted by Spencer Fricke, Christian Krueger, and Emmanuel Contreras Guzman at the University of Wisconsin Madison

THE GOAL:

With the push to get people on board using HTTP/2 over HTTP/1.1 there is some uncertainty about the most optimal way to package your website. You may be aware it was a common hack to concat and minify all your javascript into a single file and send it over, but this defeats the purpose of caching the files. You also don't see any performance gain from bundling the files together. This is where the real questions comes in:

  • If my site contains 1 MB worth of assets (JavaScript, CSS, images, etc.) is it better to send it as:
    • 1 file at 1 MB
    • 1024 files at 1 KB each
    • Somewhere in between
  • If the answer is somewhere in between, is it a predictable value?
  • Is the order in your HTML referencing of the different sized files make a difference?
  • Is any of this dependent on the server framework being used? (Nginx vs Apache)
  • Is any of this dependent on the browser being used? (Chrome vs Firefox)
  • Is the amount of latency matter? (Ethernet vs Wi-Fi vs 3G/4G)
